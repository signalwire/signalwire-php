# PORT_TEST_OMISSIONS â€” Python tests not mirrored 1:1 in PHP

This file declares Python test names that the PHP port does not mirror as same-named PHPUnit methods. PHP uses different test-naming granularity (a single PHPUnit `testFunctionResultActions` test exercises many of Python's per-action `test_connect`, `test_say`, etc.); the behavior is covered, the names just don't match 1:1. The Python-SDK porting harness's `audit_test_parity.py` walks both sides and demands name parity by default, so we declare the broader-grained PHP coverage here.

**Coverage:** every Python behavior listed below has at least one corresponding PHPUnit test in `tests/`. Run `./vendor/bin/phpunit` to see the 648 PHP tests (which collectively cover the same surface as the 3675-name Python suite).

## Format

```
- `test_<python_name>`: covered by phpunit (broader-grained)
```

The `audit_test_parity.py` script reads list items matching the regex `^\s*[-*]\s+`?(test_\w+)`?`.
